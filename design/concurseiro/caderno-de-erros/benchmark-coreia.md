## 1. Contexto Cultural — O que é o 오답노트

[confirmado] O 오답노트 (오답 = resposta errada; 노트 = caderno) é uma prática central na cultura de estudos coreana. Diferente de um simples "gabarito comentado", o 오답노트 é um diário de diagnóstico: o estudante não apenas registra a resposta correta, mas documenta *por que* raciocinou errado. É o equivalente ao que chamamos de "caderno de erros" no Brasil, porém com nível de estruturação muito maior.

Fonte: SubLearn Korean Vocabulary — <https://sublearn.com/learn/ko/vocabulary/wrong-answer-note>

**Prática física clássica (hagwon):** cortar a questão da prova/apostila e colar no caderno; ao lado, escrever em caneta colorida o raciocínio correto, a armadilha que fez errar, e o conceito faltante. Papelarias vendem cadernos com colunas pré-formatadas: "Questão / Minha Resposta / Resposta Correta / Causa do Erro".

### 2.2 Archivos de Notas / Oknote (오케이노트) — modelos de produção B2B [confirmado]

**Fonte:** Naver Blog oknote — <https://m.blog.naver.com/oknote/222162454800>  
(fabricante de cadernos personalizados para academias/hagwons coreanos)

**Tipos de layout (속지 = folha interna) documentados:**

#### Tipo A — ONE on one (1 problema por página)

**A1 — Básico:**
- Cabeçalho: data + nome do livro/prova (simples, 1 linha)
- Corpo livre: estudante organiza como quiser

**A2 — Com campo de causa do erro:**
- Cabeçalho: data + fonte (교재명/prova)
- Corpo: problema + espaço para resposta incorreta
- Campo destacado: "틀린 이유" (Causa do erro)
- Resposta correta + processo de resolução

**A3 — Com problema semelhante (유사문제):**
- Igual ao A2 + área na parte inferior para resolver 1 problema do mesmo tipo
- Finalidade: fixar o padrão e não apenas memorizar a resposta

#### Tipo B — TWO on one (2 problemas por página)

**B1 — Básico 2 em 1:**
- Cabeçalho da página: número da página + unidade/capítulo
- 2 colunas: problema + resposta (lado a lado ou dividido verticalmente)

**B2 — Com área de conceito-chave:**
- Igual ao B1 + seção inferior "핵심 내용" (Conceito Central)
- Separa o conteúdo pontual do erro do conceito estrutural

**B3 — Detalhado múltiplos campos:**
- Página dividida em 2 problemas verticais
- Cada problema tem: Problema | Motivo do Erro | Partes Esquecidas | Conceito
- Mais campos = mais diagnóstico, menos espaço para resolução

### 2.5 Template de Universidade / 서울대생 método Cornell adaptado [confirmado]

**Fonte:** Jinron News — <https://www.jinron.kr/news/articleView.html?idxno=3628>  
(perfil de estudante top do SNU/서울대 usando método Cornell adaptado para 오답노트 diário)

**Estrutura (Cornell Note modificado para osdap):**

```
+------------------+----------------------------------------+
| ABA ESQUERDA     | CORPO PRINCIPAL                        |
| (caneta vermelha)|                                        |
|                  | Data (canto superior direito)          |
| 오늘의 오답      | Disciplina específica (ex: 확률과통계  |
| (Erros do dia)   | não "수학")                            |
| — qtd por matéria|                                        |
|                  | Para cada erro:                        |
| 종합             | - Não copia o problema inteiro         |
| (Avaliação geral)| - Anota APENAS: conceito perdido,      |
|                  |   ponto crítico, armadilha específica  |
|                  | - Ex: "선지 정독하기: 키워드 찾기     |
|                  |   싸움이 아니다" (Leia os itens com   |
|                  |   atenção: não é jogo de palavras-chave)|
+------------------+----------------------------------------+
```

**Diferencial:** registra erros por TODOS os itens que gerou dúvida, não apenas os errados. Inclui "맞혔지만 찍어서 맞은 문제" (acertou por chute — esses também entram).

## 3. Padrões de Layout Recorrentes

### 3.1 Campos mínimos universais (aparecem em TODOS os formatos)

Estes campos aparecem em 100% dos templates analisados, físicos ou digitais:

1. **출처 / 날짜** — Fonte (livro, simulado, ano de prova) + Data do erro
2. **문제** — A própria questão (colada, fotografada, ou resumida em 1 frase)
3. **틀린 이유** — Causa do erro (campo mais valorizado; sem ele, o caderno "não funciona")
4. **정답 + 풀이** — Gabarito + processo de resolução correto

### 3.2 Taxonomia de causas do erro (오답 원인 분류) [confirmado]

Há consenso amplo entre blogs de estudo coreanos e academias sobre 4–5 categorias:

| Código | Nome | Tradução | Descrição |
|--------|------|----------|-----------|
| C1 | 개념 부족 | Conceito insuficiente | Não sabia o conteúdo |
| C2 | 계산 실수 | Erro de cálculo | Sabia, mas errou na conta |
| C3 | 문제 해석 오류 | Erro de interpretação | Leu errado, pegou negação, etc. |
| C4 | 시간 부족/실수 | Falta de tempo / descuido | Pressa, desatenção |
| C5 | 심리적 요인 | Fator psicológico | Ansiedade, insegurança |

Alguns sistemas usam códigos C1/C2/C3 com sub-códigos (C1a, C1b...) para diagnóstico fino. [a confirmar] — depende do nível do método.

### 3.3 Módulo de revisão espaçada (복습 주기) [confirmado]

Padrão consolidado no ecossistema coreano: curva de Ebbinghaus aplicada ao caderno de erros.

- 1ª revisão: no mesmo dia (logo após errar)
- 2ª revisão: 1 dia depois
- 3ª revisão: 3–7 dias depois
- 4ª revisão: 14 dias depois
- 5ª revisão: 30 dias depois

Nos formatos físicos: caixas de check ("N회독" = N-ésima leitura) embutidas no cabeçalho da ficha.  
Nos formatos digitais (Notion): campos de fórmula calculam automaticamente a data de cada rodada.

### 3.4 Variantes de layout por tipo de conteúdo

| Disciplina | Estrutura preferida | Observação |
|------------|---------------------|------------|
| Matemática | 1 problema/página; espaço grande para cálculo | "다시 풀기" (resolver de novo) é obrigatório |
| Linguagens | Resumo do erro em 1 frase; sem copiar texto longo | Foco no padrão, não no texto |
| Ciências | Categorização por unidade; diagrama + fórmula | Requer espaço para esquema |
| Todos | Campo de "유사문제" (problema semelhante) opcional | Alta correlação com redução de reincidência |

### 3.5 Hierarquia visual nos formatos físicos

- Cor vermelha para campos de erro/causa (destaque imediato)
- Cor azul/verde para gabarito (solução positiva)
- Stickers/símbolos: ⭐ = problema muito difícil; ✓ = revisado; 💡 = conceito fundamental
- Papel de alta gramatura (80–100g) = padrão premium Motemote/marcas de papelaria de estudo

## 6. Teardown — Interpretação e Síntese

### O que o 오답노트 coreano faz de diferente

**a) Diagnóstico do raciocínio, não do conteúdo**

O campo mais valorizado não é "gabarito" — é "틀린 이유" (causa do erro). A lógica coreana é: o estudante que sabe o gabarito mas não sabe *por que* errou vai repetir o erro. O caderno é um espelho metacognitivo. Isso ressoa diretamente com o que o módulo Caderno de Erros do Concurseiro precisa entregar.

**b) Taxonomia de causas como sistema diagnóstico**

Os 4–5 códigos de erro (C1 conceito / C2 cálculo / C3 interpretação / C4 descuido / C5 psicológico) funcionam como um sistema de triagem. Ao revisar o caderno, o estudante vê padrões: "80% dos meus erros são C3 — estou lendo rápido demais". Isso converte o caderno de repositório em ferramenta de melhoria comportamental.

**c) Revisão espaçada integrada ao layout físico**

O campo "N회독" (N-ésima leitura) da Motemote e os campos de data da Yesform não são acessórios — são o mecanismo que garante que o caderno seja usado, não só preenchido. Sem gatilho de revisão, o caderno morre na gaveta. Isso é um diferencial de produto, não apenas de método.

**d) "Resolver de novo" como campo obrigatório (matemática)**

Para exatas, o 오답노트 coreano exige espaço para resolver novamente na ficha — não só ler o gabarito. A pedagogia é: entender sem resolver não fixa. Os layouts tipo A3 (com problema semelhante) vão além: pedem um problema do mesmo tipo, não o mesmo problema.

**e) Compacidade e portabilidade**

Formatos A5 e B6 dominam o físico; o estudante carrega o osdapnote junto com o material de estudo e anota imediatamente após errar. Os formatos maiores (A4+) são para matemática/problemas longos. A Motemote usa o landscape/spring para maximizar área de escrita sem aumentar o formato.

## 7. Ideias Acionáveis para a Ficha Concurseiro

### Ideia 1 — Campo de triagem com código de erro pré-impresso

Inspiração: sistema de 5 codes (C1–C5) dos blogs coreanos

Na ficha do Concurseiro, incluir uma linha de checkboxes com os tipos de erro mais comuns em concursos públicos brasileiros:

```
CAUSA DO ERRO:  [ ] Conceito  [ ] Interpretação  [ ] Distrator  [ ] Desatenção  [ ] Tempo
```

A marcação rápida durante a correção dispensa texto longo e cria dados para análise de padrão ao final do ciclo.

### Ideia 2 — Campo "N revisões" embutido no cabeçalho da ficha

Inspiração: campo "N회독 체크 칸" da Motemote

Uma fileira de pequenos quadrados no canto superior direito da ficha (ex: 4–5 quadradinhos) que o estudante marca a cada revisão. Isso transforma cada ficha num mini-rastreador de revisão espaçada — sem precisar de app ou tabela separada.

```
Revisões: □ □ □ □ □    Data: ___/___/___
```

### Ideia 3 — Seção "Raciocínio Errado" com espaço explícito (antes da solução correta)

Inspiração: prática dominante no ecossistema coreano — registrar o raciocínio incorreto ANTES de escrever a solução correta

A ficha brasileira tende a pular direto para a solução. O modelo coreano força a reconstrução do pensamento errado — "o que eu pensei que estava certo" — o que acelera o reconhecimento do padrão quando a situação volta. Campo sugerido: "O que eu estava pensando:" (3–4 linhas) → "O que estava errado nesse raciocínio:" (1–2 linhas) → "Solução correta:".
